Cette partie va permettre de lister les différents points positifs et négatifs de Gun

Les avantages de Gun

  • Pas de serveurs

    En effet, comme Gun est décentralisé, puisque utilisant le P2P, il n'est pas nécessaire d'avoir de serveurs fixes quelque pars dans le monde (excepté les relay peers).

  • Données en “temps réel”

    L'utilisation des websockets permet la modification "en chaine" des données. Cela veut dire que si quelqu'un modifie un nœud, toutes les personnes utilisant le nœud seront averties

  • Sécurité des données grâce à SEA

    SEA permet d'avoir un très bon chiffrement des données contenues dans les utilisateurs. Les protocoles utilisés sont robustes et permettent d'être sûr (pour le moment) que leis données sont sécurisées

  • Simple d’utilisation

    Comme montré dans la partie "Utilisation" ou "Exemple", l'utilisation de Gun est très intuitive. Le système d'utilisateur avec une "inscription" et une connection est connue de tous. De plus, la navigation dans le graphe se fait aisément. La simplicité d'utilisation est l'un des points essentiels pour le créateur de Gun

Les désavantages de Gun

  • Les données dans le Public Space peuvent être modifiées par tout le monde

    En effet, ces données sont d'une part consultable par tout le monde, mais elles sont aussi modifiables par tout le monde. Dans l'exemple d'un site communautaire où tout le monde pourrait écrire des sujets, cela serait embêtant qu'une personne détruise le travail d'un autre. Il est donc nécessaire d'avoir un utilisateur pour enregistrer des données "privées".

  • Peut faire peur à cause du côté P2P

    La mension du Peer2Peer peut encore faire peur à certaines personnes, surtout si celle-ci sont mal renseignées et ont des préjugés (piratage de jeux,...)
